Input data

To run the pipeline you need to input a catalog of mutations in a cohort of tumors. The input can be provided in three different format (Tabulated, VCF and MAF). Additionally you can provide a clinical annotation file with clinical features of the samples, which will be used when exploring the results in the browser.

Tabulated format

The default format is a simple tab-separated format, containing six required columns:

  1. chromosome - just the name or number, with no 'chr' prefix
  2. start
  3. end
  4. strand - defined as + (forward) or - (reverse).
  5. allele - pair of alleles separated by a '>', with the reference allele first
  6. sample_id - Identifier of the sample.

Example:

9    134853570   134853570    +    T>C    P66

3    159866794   159866794    +    A>G    P72

22   36358654    36358654     +    G>T    P1

2    233420510   233420510    +    G>C    P52

13   19615309    19615309     +    C>T    P18

X    70360607    70360607     +    G>T    P55

13   19661627    19661627     +    C>T    P9

1    226413328   226413328    +    G>A    P66

7    99364644    99364644     +    C>T    P73

3    33074745    33074745     +    C>G    P55

IntOGen Pipeline accepts both hg18 and hg19 coordinates. The genomic coordinates system must be specified at the beginning of the pipeline.

VCF (Variant Call Format)

TCGA VCF 

VCF 4.1

VCF is a text file format for storing and reporting genomic sequence variations.

IntOGen Pipeline accepts this format and the data can be sent through the pipeline in two ways:

  1. Input a single .vcf file with sequence variants from several samples: This file will contain data from several samples in vcf format appended. Each sample can be identified by adding the header “##INDIVIDUAL=sample_id” before each of the samples data.
  2. Input a compressed file (zip/tar/tar.gz) containing several .vcf files: The compressed file must contain one vcf file for each sample. In this case, if the header “##INDIVIDUAL=sample_id” is included, this id will be used, otherwise, the name of the file will be used as sample_id.

MAF (Mutation Annotation Format)

Mutation Annotation Format (MAF) Specification

The MAF format is a tab-separated text. The file can contain many columns as it can be seen in the specification, however the only columns that are necessary for IntOGen Pipeline are:

  1. Chrom
  2. Start_Position
  3. End_Position
  4. Strand
  5. Tumor_Seq_Allele1
  6. Tumor_Seq_Allele2
  7. Tumor_Sample_Barcode

Similarly as it was explained for VCF files, MAF files are accepted in the pipeline also in two ways:

  1. Input a single .maf file with several samples
  2. Input a compressed file (zip/tar/tar.gz) containing several .maf files

Clinical annotation file

The clinical annotation file is optional, it is not really needed to run the pipeline. However it is useful to navigate your data in the browser.

The clinical annotation file should be a tabulated file that contains the ID of the sample in the first column and a number of additional columns, one for each clinical annotation. The first row is used as a header.
